TraceNet AI — Update: Expanded Apify Actors + Modern Scanner UI + Rich PDF
==========================================================================

BACKEND
-------
1. New Apify scraping connectors (app/connectors/apify_scrapers.py):
   - apify_instagram   -> apify/instagram-profile-scraper (public profile + posts)
   - apify_tiktok      -> clockworks/tiktok-profile-scraper (public profile + videos)
   - apify_twitter     -> apidojo/twitter-user-scraper (public profile + tweets)
   - apify_linkedin    -> apimaestro/linkedin-profile-detail (public profile; url/username)
   - apify_reddit      -> trudax/reddit-user-scraper (public user + posts)
   - apify_contact_info-> apify/contact-info-scraper (public email/phone on a domain)
   - apify_domain_whois-> epctex/whois-scraper (public WHOIS record for a domain)
   All are public-source only. No login / OTP / contact-sync / private enumeration.
   Registered in app/connectors/registry.py; each toggleable from .env.

2. public_source_agent now derives bare domains from URLs + emails and passes a
   "domain" bucket to the connector registry so WHOIS/contact actors fire cleanly.

3. config.py + .env / .env.example: new ENABLE_* flags, actor IDs, and
   APIFY_PROFILE_SCRAPER_MAX_ITEMS / _MAX_POSTS.
